# Template Method

Template Method ― 具体的な処理をサブクラスにまかせる

## メリット

- スーパークラスにロジックが集中しているので、バグが生じてもTemplateMethodのみを修正すればいい
- サブクラスの設計が簡潔になる

## デメリット

- サブクラスの数が増加する
- 親と子の関係が密接なので、スーパークラスのソースを確認しないと実装が難しい
- スーパークラスの処理が大きくなると、サブクラスの自由度が減少する

## 登場人物

特になし
